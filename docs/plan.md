# Project Plan — Jacobs Feed & Supply Website

## Overview
This plan will outline how the Jacobs Feed & Supply website project will be executed, including the risks and a basic work schedule. The plan is based on project management concepts from *Project Management for the Unofficial Project Manager (PMUPM)*.

---

## Risk Analysis (TAME Framework)

### Risk 1: Scope Creep (adding too many pages/features)
- **T (Transfer):** Not applicable (scope is owned by the project).
- **A (Avoid):** Define clear deliverables and keep the site limited to two pages.
- **M (Mitigate):** Track “nice-to-have” features in a notes section but do not implement them.
- **E (Embrace):** If time allows at the end, add small improvements that do not change the scope (minor layout polish).

### Risk 2: Deployment Issues with GitHub Pages
- **T (Transfer):** Use GitHub Pages documentation and a clean repo structure.
- **A (Avoid):** Keep file names simple and in the root directory (index.html at root).
- **M (Mitigate):** Test the site locally first, then verify links after deployment.
- **E (Embrace):** Treat errors as a learning opportunity; document the fix if issues occur.

### Risk 3: Time Management / Outside Responsibilities
- **T (Transfer):** Not applicable.
- **A (Avoid):** Start documentation early and break tasks into smaller steps.
- **M (Mitigate):** Use a short checklist and complete the prototype first, then refine writing.
- **E (Embrace):** Use limited time as a reason to keep the site clean and focused.

### Risk 4: Broken Navigation / Inconsistent Styling
- **T (Transfer):** Not applicable.
- **A (Avoid):** Use one shared stylesheet and duplicate the same header/nav on both pages.
- **M (Mitigate):** Click-test every link before submission.
- **E (Embrace):** If something breaks, fix it quickly and note the lesson learned for future projects.

## Work Breakdown Structure (WBS) — Task Schedule

1. **Project Setup**
   - Create GitHub repository
   - Add folder structure (`docs/`)

2. **Draft Documentation**
   - Write/revise `docs/scope.md`
   - Create `docs/plan.md` with TAME + WBS

3. **Build Website Prototype**
   - Create `index.html` (semantic structure + nav)
   - Create `about.html` (semantic structure + nav)
   - Create `style.css` (shared styling)

4. **Testing**
   - Open pages locally
   - Confirm navigation works both ways
   - Check mobile responsiveness (basic)

5. **Finalize Repository**
   - Write `README.md` (links, purpose, overview)
   - Write `docs/retrospective.md`

6. **Deploy**
   - Enable GitHub Pages
   - Verify live link works and matches README

7. **Final Review**
   - Confirm all required files exist
   - Confirm repo is public
   - Submit the repository link
