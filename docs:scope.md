# Project Scope Statement

## Project Title
**Jacobs Feed & Supply – Website Development Project**

---

## Project Purpose / Justification
The purpose of this project is to design and develop a simple, informative website for a fictional small business called **Jacobs Feed & Supply**, an animal and livestock feed store.

I chose this topic because feed stores play an important role in rural communities, especially for livestock owners, farmers, and pet owners. Many small businesses like this rely heavily on word of mouth, but having a basic website makes it much easier for customers to find store hours, product information, and contact details quickly.

This project allows me to apply project management principles from Chapter 3 of PMUPM by clearly defining the scope, deliverables, and boundaries before beginning development. By planning everything out first, I can avoid scope creep and stay focused on building exactly what is required.

---

## Project Description
This project involves creating a multi-page informational website using semantic HTML and basic CSS styling. The website will focus on clearly presenting information about the store and what it offers.

The website will include:
- A **Home** page introducing Jacobs Feed & Supply
- A **Products** page describing livestock feed, pet food, and farm supplies
- A **Services** page (custom feed orders, delivery options, bulk pricing, etc.)
- A **Contact** page with store hours, location, and a contact form
- A consistent navigation menu linking all pages together

The final product will be a static website hosted on **GitHub Pages**.

---

## Project Objectives (SMART Goals)
1. Develop a **4-page static website** using semantic HTML elements.
2. Ensure all navigation links function properly between pages.
3. Apply clean, basic CSS styling to improve readability and layout.
4. Successfully host the website on GitHub Pages before the final deadline.
5. Keep the design simple, organized, and professional.

---

## Deliverables
The project will produce the following deliverables:

- `index.html`
- `about.html`
- `style.css`
- A GitHub repository containing all files
- A publicly accessible GitHub Pages link
- Any required final documentation or reflection

---

## High-Level Requirements
- Use semantic HTML elements (`header`, `nav`, `main`, `section`, `footer`)
- Include working navigation across all pages
- Include at least one relevant image
- Maintain organized file structure
- Deploy the site using GitHub Pages
- Stay within the defined project scope

---

## Boundaries (In Scope / Out of Scope)

### In Scope
- Informational content about the business
- Basic CSS styling
- Static website pages
- A simple contact form (non-functional placeholder)
- Simple responsive layout

### Out of Scope
- Online ordering or e-commerce functionality
- Payment processing
- Customer accounts or login systems
- Inventory tracking systems
- Backend databases
- Advanced JavaScript features

This project will remain a static informational website only.

---

## Constraints
- Limited to beginner-level HTML and CSS
- Must be completed within the semester timeline
- No backend or server-side programming
- Must follow assignment and PMUPM Chapter 3 guidelines

---

## Assumptions
- Jacobs Feed & Supply is a fictional local business serving a rural community.
- Customers mainly use the website for information, not online purchases.
- GitHub Pages will be used successfully for hosting.
- The project will be developed using Visual Studio Code.

---

## Risks
- Issues with GitHub deployment
- Broken navigation links
- Adding unnecessary features beyond the original plan
- Time management challenges

To reduce these risks, I will test links carefully and stick closely to the defined scope.

---

## Success Criteria
The project will be considered successful if:

- All four pages are completed and properly linked
- The site is publicly accessible through GitHub Pages
- The project remains within scope
- All assignment requirements are met
- The website looks clean, organized, and professional
